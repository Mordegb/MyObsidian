#programação 
 
como dar commit com resumo e descrição: 
```
git commit -m "Assunto" -m "Descrição..."
```

- o resumo do commit deve ter ate 50 caracteres -> ser objetivo
- o corpo do commit deve ter até 72 caracteres por linhas, usando quantas linhas quiser  

Tipos de commit completos:
- <span style="color:rgb(174, 0, 255)">Feat</span>: uma nova feature (recurso) que você está adicionando a uma aplicação específica 
- <span style="color:rgb(174, 0, 255)">Fix</span>: a resolução de um bug
- <span style="color:rgb(174, 0, 255)">Style</span>: recurso e atualizações relacionadas à estilização e nome de variáveis ou de funções.

- <span style="color:rgb(174, 0, 255)">Refactor</span>: refatoração de uma seção específica da base de código
- <span style="color:rgb(174, 0, 255)">Test</span>:tudo o que for relacionado a testes
- <span style="color:rgb(174, 0, 255)">Doc</span><span style="color:rgb(174, 0, 255)">s</span>: tudo o que for relacionado à documentação
- <span style="color:rgb(174, 0, 255)">Chore</span>: manutenção regular do código.
